Recursive Functions

Recursive functions are functions that achieve their intended purpose by calling themself, inside the function. Recursive funcitons consist of two parts:

  1. Base Case
  2. Step to reach the base case

The JavaScript "Promise" Constructor

The JavaScript promise constructor is used to make a promise to complete a task at a later time. The promise constructor takes a function as an argument and has two outcomes, resolve and reject. Resolve and reject are methods passed into the function as parameters. Take a look at this example:

const sendData = new Promise((resolve, reject) => {
 // use resolve and reject methods to return message based on whether or not data was sent successfully.
 let dataSent;
 if (dataSent) {
  resolve("Data was sent successfully.");
 } else {
  reject("Error - Data not sent.");
 }
});

Regular Expressions - "Regex"

Regular Expressions, also known as Regex, is a syntactic sugar used by developers to search for characters in a string. Here we will go over the basics, but the more in depth you go the more versatile your string searches can become! Regex is used by putting the string that you're searching for between two forward-slashes (/searchterm/). Let's start with an example:

// assign some string to a variable
let myStr = "Here is some text. There is some text.";
// assign the regex to a variable. This will search for 'text' when applied to astring.
let myRegex = /text/;

Now we have the string that we will be searching through, myStr, and the string we will be searching for, myRegex. So how do we apply the regex to the string? There are two methods:

  1. test - returns true if regex is present in string, false otherwise.
  2. match - returns an array with the first match to the regex.

Let's look at an example of test:

// carry over variable delaractions from last example
let myStr = "Here is some text. There is some text.";
let myRegex = /text/;
// test example
let result = myRegex.test(myStr);
console.log(result);

In this case, the console will show

true

, since the test method found 'text' within the string, myStr. Now let's look at an example of match:

// carry over variable delaractions from last example;
let myStr = "Here is some text. There is some text.";
let myRegex = /text/;
// match example
let result = myStr.match(myRegex);
console.log(result);

In this case, the console will show

[text]

, returning the first value of 'text' in the string. So how can we return an array with all the matches of 'text' in our string? For that, we must use flags. If you're familiar with console commands, you probably have an idea of flags. Unlike console commands, these are added to your JavaScript code. To return all values in the string that match the regex, use the 'g' flag. It looks like this:

// run the same code as last time with the g flag
let myStr = "Here is some text. There is some text.";
let myRegex = /text/g;
let result = myStr.match(myRegex);
return result;

Notice the 'g' at the end of the myRegex declaration. This code will now return all instances of 'text' within myStr, like this:

[text, text]
